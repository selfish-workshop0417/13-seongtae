# 구독박스 (Subbox) — PRD

## 1. 서비스 개요
구독박스(Subbox)는 AI 툴, OTT, 음악 스트리밍 등 늘어나는 구독 서비스를 한 곳에서 관리할 수 있는 웹 기반 도구다. 로그인 없이 로컬 저장 방식으로 개인 정보 부담 없이 사용할 수 있으며, 개인 사용자부터 소규모 팀·스타트업까지 구독 비용과 종료일을 한눈에 파악하고 알림을 받을 수 있는 것이 핵심 가치다.

## 2. 폴더 구조
이 레포는 5개 팀(파트)에 따른 5개 영역으로 나뉜다:
- planning/ — 기획 (무엇을 만들 것인가)
- design/ — 디자인 (어떻게 보여줄 것인가)
- cx/ — CX (누가 어떻게 사용할 것인가)
- marketing/ — 마케팅 (어떻게 알릴 것인가)
- strategy/ — 전략 (어떻게 지속 가능하게 만들 것인가)

각 영역의 상세는 해당 폴더에서 작성된다.

## 3. 기획 (Planning)
- 구독 서비스 등록: 이름, 가격, 결제 주기, 종료일, 카테고리 입력
- 대시보드: 전체 구독 목록 + 월 총 지출 합산 표시
- 알림: 종료 3일 전 / D-day 브라우저 푸시 + 카카오톡 알림
- 카테고리 필터: AI, OTT, 음악, 업무툴, 기타
- MVP 범위: 구독 등록·조회·삭제 + 브라우저 푸시 알림

상세는 `planning/` 폴더 참조. [작성 예정]

## 4. 디자인 (Design)
심플하고 정보 밀도가 높은 대시보드 스타일로, 카드 UI로 각 구독 서비스를 시각화한다. 종료일이 임박한 항목은 색상으로 강조 표시해 한눈에 파악할 수 있게 한다.
주요 화면 흐름: 대시보드(구독 목록 + 총 지출) → 구독 추가 모달 → 상세 편집 → 알림 설정

상세는 `design/` 폴더 참조. [작성 예정]

## 5. CX (Customer Experience)
타겟 페르소나: AI 툴 3개, OTT 2개, 음악 1개 등 월 6개 이상 구독을 쓰는 20~40대 직장인 / 구독 비용을 추적하고 싶은 스타트업 팀장.
사용자 여정의 핵심 단계: 사이트 접속 → 구독 추가 → 대시보드에서 현황 파악 → 알림 설정 → 종료일 전 알림 수신 → 구독 연장 or 해지 결정

상세는 `cx/` 폴더 참조. [작성 예정]

## 6. 마케팅 (Marketing)
첫 사용자 확보: AI 툴 커뮤니티(CLOVA, ChatGPT 카페), 개발자 커뮤니티(Disquiet, 인디해커 한국판), X(트위터) 등 구독 피로를 느끼는 사람들이 모인 채널을 중심으로 바이럴.
주요 채널: X/트위터 (스크린샷 바이럴), Disquiet 제품 런칭

상세는 `marketing/` 폴더 참조. [작성 예정]

## 7. 전략 (Strategy)
현재는 무료 로컬 저장 방식으로 진입 장벽을 낮춘다. 이후 팀 공유 기능(서버 동기화)을 유료 플랜으로 전환하는 Freemium 모델을 검토한다.
성장 방향: 개인 → 팀 플랜 → API 연동(구독 서비스 자동 인식) 순으로 확장
